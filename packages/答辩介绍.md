# 🏨 HotelPlatform · 酒店预订管理系统

> 携程前端训练营结业项目 · 2026.03

---

## 一、项目概述

本项目是一个**仿携程风格的酒店预订管理系统**，分为三端协同开发：

| 端              | 定位               | 负责人 |
| -------------- | ---------------- | --- |
| **Mobile 移动端** | 面向住客的预订 App      | 陈俊  |
| **Admin PC 端** | 面向商家 & 平台管理的后台系统 | 唐超  |
| **Server 服务端** | 提供所有业务 REST API  | 吴民港 |

---

## 二、技术栈

| 层次       | 技术                        |
| -------- | ------------------------- |
| Mobile 端 | Vue 3 + TypeScript + Vite |
| Admin 端  | React + Vite              |
| Server 端 | Node.js + Express + MySQL |
| 文件存储     | 阿里云 OSS                   |
| 部署       | 阿里云 ECS（服务端）              |

---

## 三、分工详解

### 📱 Mobile 移动端（陈俊）

面向 C 端住客，完整实现酒店搜索到下单的预订闭环。

| 模块                     | 功能说明                 |
| ---------------------- | -------------------- |
| **首页 Home**            | 城市选择、日期选择、关键词搜索入口    |
| **城市选择 CitySelect**    | 支持定位城市、热门城市、字母索引快速切换 |
| **搜索 Search**          | 关键词模糊搜索酒店            |
| **酒店列表 List**          | 筛选（价格/评分/星级）、排序、分页展示 |
| **酒店详情 Detail**        | 酒店图文介绍、房型浏览          |
| **填写订单 OrderFill**     | 入住人信息填写、价格确认         |
| **模拟支付 PaymentResult** | 支付结果页面               |
| **我的订单 OrderList**     | 查看个人历史订单及订单状态        |
| **个人中心 User**          | 账号信息、登录退出            |

---

### 🖥️ Admin PC 端（唐超）

双角色后台系统，**商家（Merchant）** 管理自己的酒店和订单，**管理员（Admin）** 负责平台审核与统计。

**商家端功能：**

| 模块                | 功能说明                     |
| ----------------- | ------------------------ |
| **工作台 Dashboard** | 今日数据概览（订单数、收入、入住率）       |
| **酒店管理 Hotels**   | 新增 / 编辑 / 下架酒店，上传图片至 OSS |
| **房型管理 Rooms**    | 新增 / 编辑 / 删除房型，管理房间号列表   |
| **订单管理 Orders**   | 查看订单列表、确认订单、分配具体房间号      |

**管理员端功能：**

| 模块                  | 功能说明                  |
| ------------------- | --------------------- |
| **酒店审核 HotelAudit** | 审批商家提交的酒店上架申请，可填写拒绝原因 |
| **用户管理 Users**      | 商家账号列表、启用/禁用、关键词搜索    |
| **数据统计 Statistics** | 平台整体订单量、收入等数据展示       |

---

### ⚙️ Server 服务端（吴民港）

基于 Express 构建 REST API，实现业务核心逻辑，共 **11 个路由模块**。

| 路由文件               | 接口路径前缀               | 核心职责                                                    |
| ------------------ | -------------------- | ------------------------------------------------------- |
| `loginPC.js`       | `/api/auth`          | 商家/管理员登录（账号/手机/邮箱多方式）                                   |
| `loginMobile.js`   | `/api/mobile/auth`   | C 端用户注册、登录                                              |
| `hotels.js`        | `/api/hotels`        | 商家酒店 CRUD、图片上传、酒店列表分页搜索                                 |
| `hotelsMobile.js`  | `/api/mobile/hotels` | C 端酒店搜索、筛选、详情、评价查询                                      |
| `hotelsReview.js`  | `/api/review`        | 评价发布、评价列表、评分统计                                          |
| `rooms.js`         | `/api/rooms`         | 房型 CRUD，同步维护 `hotels.room_number` 和 `room_inventory` 库存 |
| `orderMobile.js`   | `/api/mobile/order`  | C 端下单、支付确认、订单状态流转                                       |
| `orderPC.js`       | `/api/orderPC`       | 商家确认订单、分配房间号、订单详情                                       |
| `userManage.js`    | `/api/users`         | 管理员对商家账号的增删改查                                           |
| `controlPC.js`     | `/api/control`       | 酒店审核通过/拒绝，管理员权限控制                                       |
| `controlManage.js` | `/api/controlManage` | 平台数据统计汇总                    |

**核心设计亮点：**

- **订单状态机**：`待付款(1) → 待确定(2) → 待入住(3) → 已完成(4)`，每步严格校验当前状态
- **日历库存表** `room_inventory`：按房型 × 日期维护可用房间数，支撑跨日期搜索可用性
- **库存联动**：编辑/删除房型时，自动同步 `hotels.room_number` 和 `room_inventory`
- **参数化查询**：所有 SQL 使用 `?` 占位符，防止 SQL 注入

---

## 四、核心业务流程

```
[住客] 搜索城市+日期 → 列表筛选 → 查看详情 → 选择房型 → 填写订单 → 模拟支付
                                                                          ↓
[商家] 收到待确定订单 → 后台确认 → 分配具体房间号 → 订单变为"待入住"
```
